# Concepts

DataChain is built around a small number of ideas. Understanding them makes the entire API predictable. Start with Data Memory and Datasets, then explore deeper topics as needed.

- [Data Memory](data-memory.md) -- what the system remembers and why it compounds
- [Datasets](datasets.md) -- the atomic unit: named, versioned, typed, immutable
- [Chain](chain.md) -- lazy composable pipelines with optimization and atomicity
- [Files and Types](files-and-types.md) -- the File abstraction, modality types, annotation types, and the type system
- [Execution Model](execution-model.md) -- the dual-engine architecture: Python Data Engine + Memory Engine
- [Provenance](provenance.md) -- automatic lineage tracking for every dataset
- [Knowledge Base](knowledge-base.md) -- navigable discovery surface for humans and agents
